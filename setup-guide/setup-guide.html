1. アプリケーションパスワードとは？

アプリケーションパスワードは、WordPress 5.6以降で使える機能です。 通常のログインパスワードとは別に、アプリ専用のパスワードを作成できます。

メリット

  • 通常のログインパスワードを入力せずに済む
  • いつでも無効化できる（通常のログインには影響なし）
  • アプリごとに別のパスワードを使える

この拡張機能では、アプリケーションパスワードを使用してWordPressと安全に接続します。 通常のログインパスワードは使用しないため、セキュリティ面でも安心です。

2. アプリケーションパスワードを有効にする

WordPress 5.6以降をお使いの場合、この機能はデフォルトで有効になっています。

確認方法

WordPress管理画面にログインし、「ユーザー」→「プロフィール」を開きます。 ページを下にスクロールすると「アプリケーションパスワード」というセクションがあれば有効です。

WordPress管理画面 → ユーザー → プロフィール
「アプリケーションパスワード」セクションの位置

セクションが表示されない場合

  • サイトがHTTPSに対応しているか確認してください
  • セキュリティプラグイン（Wordfenceなど）が無効化している可能性があります
  • WordPress 5.6より古いバージョンを使用している可能性があります

3. パスワードを発行する手順

  1. WordPress管理画面にログイン

    ブラウザで以下のURLにアクセスしてログインします：

    https://あなたのサイト.com/wp-admin/
  2. 「ユーザー」→「プロフィール」を開く

    左側のメニューから「ユーザー」をクリックし、「プロフィール」を選択します。

  3. 「アプリケーションパスワード」セクションを探す

    ページを下にスクロールして「アプリケーションパスワード」セクションを見つけます。

  4. 名前を入力

    「新しいアプリケーションパスワード名」に以下を入力します：

    ブログ投稿アシスタント

    ※名前は何でもOKですが、後で何のパスワードか分かるようにしておきましょう。

  5. 「新しいアプリケーションパスワードを追加」をクリック

    ボタンをクリックするとパスワードが生成されます。

  6. パスワードをコピー

    表示されたパスワードをコピーします。

    xxxx xxxx xxxx xxxx xxxx xxxx

重要: パスワードは一度しか表示されません！

この画面を閉じると、パスワードは二度と表示されません。 必ずコピーしてから画面を閉じてください。

もしコピーし忘れた場合は、そのパスワードを「無効化」して、 新しいパスワードを発行してください。

4. サイトのURLを確認する

正しいURLの例

https://example.com
https://blog.example.com

間違ったURLの例

https://example.com/wp-admin/ ← /wp-admin/ は不要
http://example.com ← HTTPSを使用してください

確認方法

ブラウザでサイトのトップページを開いたときのURLです。 管理画面（/wp-admin/）のURLではありません。

5. 拡張機能の使い方

  1. ChatGPT・Claude・Geminiでブログ記事を作成
    普段通りAIに記事を書いてもらいます。
  2. 出力形式を指定
    記事の最後に「```markdownで囲んで出力してください」と伝えます。
  3. 投稿ボタンをクリック
    マークダウンコードブロックの右下に「📤 WPに投稿」ボタンが表示されます。
  4. 確認して投稿
    タイトルやカテゴリを確認し、「投稿する」をクリックで完了！

6. よくある質問

「認証に失敗しました」と表示される

ユーザー名が正しいか確認してください（メールアドレスではなくユーザー名です）。 また、通常のログインパスワードではなく、アプリケーションパスワードを使用しているか確認してください。 パスワードはスペースを含めてそのままコピーしてください。

「WordPressが見つかりません」と表示される

URLが正しいか確認してください。https:// で始まっているか、末尾に余計な文字（/wp-admin/ など）がついていないか確認してください。 また、WordPressのREST APIが有効になっているか確認してください。

「投稿権限がありません」と表示される

WordPressでのユーザー権限を確認してください。 「投稿者」以上の権限が必要です。 WordPress管理者に権限を確認してもらってください。

アプリケーションパスワードのセクションがない

WordPress 5.6より古いバージョンを使用している可能性があります。 または、セキュリティプラグインが機能を無効化している可能性があります。 サイトがHTTPSに対応しているかも確認してください。

投稿ボタンが表示されない

マークダウンコードブロック（```markdown で始まるコード）があるか確認してください。 また、記事の内容が短すぎる場合（200文字未満）は検出されないことがあります。
